# Центральный vs per-project Dolt серверы

## Симптомы

beads-web показывает 0 beads для проекта, хотя `bd list` в терминале работает. Или: `bd sql` возвращает "Dolt server unreachable at 127.0.0.1:3307".

## Что изменилось

bd v0.59 перешёл с одного центрального Dolt-сервера (порт 3307) на отдельный сервер для каждого проекта.

| | Старая схема (до v0.59) | Новая схема (v0.59+) |
|---|---|---|
| Сервер | Один на порту 3307 | Свой для каждого проекта |
| Порты | 3307 | 13307–14306, хеш от пути проекта |
| Базы | Все в одном месте | Каждая в `.beads/dolt/` проекта |
| Запуск | Вручную | Автоматически при первой bd-команде |

## Как найти порт проекта

```bash
# Способ 1: файл с портом (создаётся при запуске сервера)
cat .beads/dolt-server.port

# Способ 2: bd info
bd info
# Покажет Mode: server, порт и путь к базе

# Способ 3: найти процесс
# Linux/macOS
ss -tlnp | grep dolt
# Windows
netstat -ano | findstr LISTENING | findstr dolt
```

## Частые проблемы

### "Dolt server unreachable at 127.0.0.1:3307"

Per-project сервер ещё не запущен. Запустите любую bd-команду из директории проекта:

```bash
cd /path/to/project
bd list
# Сервер стартует автоматически
```

### beads-web не видит beads проекта

beads-web подключается к центральному серверу (3307). Per-project базы на других портах для него невидимы.

Решение: beads-web v0.6+ читает `.beads/dolt-server.port` и подключается к per-project серверу напрямую. Обновите beads-web.

### metadata.json указывает на 3307, но база на другом порту

После обновления bd, `metadata.json` может содержать `dolt_mode: "server"` без указания порта. bd по умолчанию пытается подключиться к 3307.

Проверьте:

```bash
cat .beads/metadata.json
# Если нет dolt_port — bd использует авто-обнаружение
# Файл .beads/dolt-server.port содержит актуальный порт
```

### Данные есть на центральном сервере, но не на per-project

Старые проекты могли хранить базу на центральном сервере. После обновления bd создаёт новую пустую базу в `.beads/dolt/`.

Проверьте центральный сервер:

```bash
# Подключитесь к центральному и посмотрите базы
bd sql "SHOW DATABASES"
# Если ваша база там — нужна миграция
# См. jsonl-to-dolt-migration-ru.md
```

## Диагностика

```bash
# 1. Проверьте конфигурацию проекта
cat .beads/metadata.json

# 2. Проверьте запущенные Dolt-серверы
# Linux/macOS
ps aux | grep "dolt sql-server"
# Windows
tasklist | findstr dolt

# 3. Проверьте что bd видит данные
bd sql "SELECT COUNT(*) FROM issues"

# 4. Проверьте порт
cat .beads/dolt-server.port 2>/dev/null || echo "Файл порта не найден — сервер не запущен"
```
